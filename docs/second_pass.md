# Second Pass Enrichment Protocol

How to deepen an existing stock guide with additional data. Read this before performing any second pass work.

## When to Second Pass

A topic needs a second pass when:
- The page is **sparse** (<10 original tossups+bonuses, or <4 work sections)
- Important works lack their own answerline data (only mentioned in parent topic's clues)
- The topic was generated by an agent that produced shallow analysis

## Pipeline

### Step 1: Load existing analysis
Read `output/{slug}_analysis.json`. Note:
- How many work sections exist
- How many cards exist
- Which works have thin clue coverage
- What the comprehensive summary covers

### Step 2: Text mention query (if sparse)
If the page is sparse, fetch questions where the topic appears in clue text but is NOT the answer.
For sparse topics, also expand to difficulties 5-6 to catch more data:
```bash
python3 lib/run.py "TOPIC" "5,6,7,8,9,10" --mentions
```
This saves to `output/{slug}_mentions_clues.txt`. These clues provide contextual information — e.g., how other topics reference this one.

### Step 3: Subitem answerline queries
For each major work listed in the analysis (skip "General / Biographical", "Other Works"):
```bash
python3 lib/run.py "Work Name" "7,8,9,10"
```
For sparse topics, expand to difficulties 5-6 here too:
```bash
python3 lib/run.py "Work Name" "5,6,7,8,9,10"
```
Strip dates and parentheticals from work names before querying. For example:
- "The Course of Empire (1833–1836)" → query "The Course of Empire"
- "Vltava / The Moldau" → query "The Moldau" (use the more specific/English name)

If a query returns 0 results, skip it.

### Step 4: Merge into existing analysis
Read the existing analysis JSON and ALL new clue files. Merge following these rules:

**Preserve everything existing:**
- Never remove work sections, clues, cards, cross-refs, or metadata
- Never reduce frequency counts
- Never overwrite descriptions with shorter versions

**Add new data:**
- Add new clue entries to appropriate work sections
- If subitem queries reveal enough new material about a work, expand its description
- If text mentions reveal works/themes not previously covered, add new work sections (only if 3+ mentions)
- Generate cards for new clues (append to cards array)
- Rewrite `comprehensive_summary` to incorporate all information

**Track the enrichment:**
Add a `second_pass` field to the JSON:
```json
{
  "second_pass": {
    "date": "2026-03-20",
    "text_mentions_queried": true,
    "subitem_queries": ["The Oxbow", "The Course of Empire"],
    "new_clues_added": 12
  }
}
```

### Step 5: Remove from queue
```bash
python3 lib/topic_queue.py remove-second "TOPIC"
```

### Step 6: Render
```bash
python3 -c "
from render import render_html
import json
with open('output/{slug}_analysis.json') as f:
    analysis = json.load(f)
render_html(analysis, 'output/{slug}_stock.html')
"
python3 render_cards.py
python3 render_questions.py
python3 build_index.py
```

## Card Rules (same as first pass)
- One fact per card
- Indicator before colon: `Painting:`, `Novel:`, `Concept:`, etc.
- Back: `Work (Creator)` for work-specific, just `Creator` for general
- Never leak the answer on the front
- Tags: always empty `[]`

## What NOT to Do
- Don't re-derive existing clues from scratch — add to them
- Don't search for images — that's handled by `lib/fix_images.py` separately
- Don't query works that already have rich clue coverage (5+ clues) unless you suspect there's much more data
