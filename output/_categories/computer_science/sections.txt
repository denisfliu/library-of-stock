# Theory of Computation & Complexity
> Formal models of computation, complexity classes, and the limits of what can be computed.
Turing machines | Abstract computational model of a tape and read/write head, defining the limits of what can be computed.
NP | Complexity class of decision problems whose solutions can be verified in polynomial time; central to the P vs NP question.
Big O Notation | Notation describing an algorithm's asymptotic time or space complexity as input size grows.
NP-complete | Class of the hardest problems in NP, to which all other NP problems can be reduced in polynomial time.
quantum computers | Computers exploiting superposition and entanglement of qubits, promising speedups for problems like factoring.
Boolean satisfiability problem | Problem of determining whether a Boolean formula can be made true; the first problem proven NP-complete.
lambda calculus | Formal system for expressing computation via function abstraction and application, foundational to functional programming.
O | Symbol of Big O notation, describing an algorithm's asymptotic upper-bound complexity.
regular | Describes languages recognized by finite automata and expressed via regular expressions.
halting problem | Undecidable problem of determining whether an arbitrary program will terminate, proved by Turing.
Turing machine | Abstract computational model of a tape and read/write head defining the limits of computability.
time | Central resource measured in algorithmic complexity analysis, as in time complexity.
Boolean algebra | Algebraic system of true/false values and operations (AND, OR, NOT), foundational to digital logic.
complexity | Study of the time and space resources required to solve computational problems.
grammars | Formal rules defining a language's syntax, classified by the Chomsky hierarchy.
Grover's algorithm | Quantum algorithm providing a quadratic speedup for unstructured search problems.
Karnaugh maps | Diagrammatic method for simplifying Boolean algebra expressions in digital logic design.
linear | Describes complexity or time growing proportionally with input size.
n log n | Common asymptotic time complexity of efficient comparison-based sorting algorithms like mergesort.
P | Complexity class of decision problems solvable in polynomial time.
randomized | Describes algorithms that use random choices to achieve good expected performance or approximate solutions.
Shor's algorithm | Quantum algorithm efficiently factoring large integers, threatening RSA encryption.
traveling salesman problem | NP-hard problem of finding the shortest route visiting a set of cities exactly once and returning to the start.
Boolean | Data type representing true or false values, foundational to logic and conditional statements.
context-free grammars | Formal grammars generating languages recognized by pushdown automata, used to define programming language syntax.
Conway's Game of Life | Cellular automaton devised by John Conway where cells live, die, or reproduce based on neighbor rules; Turing-complete.
graph isomorphism problem | Problem of determining whether two graphs are structurally identical; its complexity status is not fully resolved.
Kleene star | Operation in formal language theory denoting zero or more repetitions of a pattern, used in regular expressions.
prime numbers | Integers greater than 1 divisible only by 1 and themselves; the hardness of factoring them underlies RSA cryptography.
SAT | The Boolean satisfiability problem, the first problem proven NP-complete via the Cook-Levin theorem.
XOR | Exclusive-or logical operation returning true when inputs differ, used in cryptography and digital circuits.

# Algorithms
> Techniques for solving problems efficiently, from sorting and searching to graph and optimization methods.
recursion | Technique where a function calls itself to solve smaller instances of a problem, requiring a base case.
matrix multiplication | Operation combining two matrices, optimized by algorithms like Strassen's beyond the naive cubic-time approach.
quicksort | Divide-and-conquer sorting algorithm using a pivot to partition an array, with average-case O(n log n) time.
Dijkstra's algorithm | Greedy algorithm finding shortest paths from a source vertex to all others in a graph with non-negative weights.
regular expressions | Patterns describing sets of strings, used for searching and matching text.
dynamic programming | Technique solving problems by breaking them into overlapping subproblems and storing solutions to avoid recomputation.
linear programming | Optimization technique maximizing or minimizing a linear objective subject to linear constraints.
depth first search | Graph traversal algorithm exploring as far as possible along each branch before backtracking.
mergesort | Divide-and-conquer sorting algorithm splitting, sorting, and merging subarrays in O(n log n) time.
sorting | Task of arranging data in a defined order, via algorithms like quicksort, mergesort, and heapsort.
finding the shortest path | Class of graph problems solved by algorithms like Dijkstra's, Bellman-Ford, and Floyd-Warshall.
memoization | Optimization technique caching results of expensive function calls to avoid redundant computation.
prime factorization | Decomposition of an integer into its prime factors; its computational hardness underlies RSA cryptography.
sequence alignment | Technique for arranging biological sequences to identify regions of similarity, often via dynamic programming.
string searching | Task of finding occurrences of a pattern within text, solved by algorithms like KMP and Boyer-Moore.
Bellman-Ford algorithm | Shortest-path algorithm handling negative edge weights, slower than Dijkstra's but more general.
maximum-flow problem | Graph problem finding the greatest possible flow from a source to a sink, solved via algorithms like Ford-Fulkerson.
Monte Carlo methods | Computational techniques using repeated random sampling to estimate numerical results.
multiplication | Basic arithmetic operation, optimized in computing via algorithms like Karatsuba's beyond schoolbook methods.
online algorithms | Algorithms that process input piece-by-piece without knowledge of the future, as opposed to offline algorithms.
search | Task of finding a target element or path, solved via methods like binary search or graph search.
searching | Task of finding a target element or path, solved via methods like binary search or graph search.
backtracking | Algorithmic technique incrementally building candidate solutions and abandoning ones that fail constraints.
binary search | Algorithm finding a target in a sorted array by repeatedly halving the search interval.
bubble sort | Simple sorting algorithm repeatedly swapping adjacent out-of-order elements, with O(n^2) worst-case time.
collision detection | Computational technique determining when two or more objects intersect, common in graphics and physics simulation.
Divide and conquer algorithms | Algorithmic strategy breaking a problem into subproblems, solving them recursively, and combining the results.
Fast Fourier Transform | Efficient algorithm computing the discrete Fourier transform in O(n log n) time, widely used in signal processing.
Floyd-Warshall algorithm | Dynamic programming algorithm computing shortest paths between all pairs of vertices in a graph.
greedy algorithms | Algorithms making locally optimal choices at each step in hopes of finding a global optimum.
knapsack problem | Optimization problem selecting items with given weights and values to maximize value within a capacity constraint.
Merge Sort | Divide-and-conquer sorting algorithm splitting, sorting, and merging subarrays in O(n log n) time.
optimization | Process of finding the best solution from a set of feasible options according to some objective.
pseudorandom number generation | Process of generating sequences of numbers that approximate randomness via deterministic algorithms.
recursive | Describes a function or process defined in terms of itself, requiring a base case to terminate.
sorting algorithms | Methods for arranging data in order, including quicksort, mergesort, and heapsort.

stacks | Last-in-first-out structures supporting push and pop operations.
tree | Hierarchical structure of nodes with a root and children.

hash collisions | Occur when two keys map to the same slot in a hash table, resolved via chaining or open addressing.

hash tables | Structures mapping keys to values via a hash function, offering average constant-time lookup.

functional programming languages | Languages like Haskell and Lisp emphasizing immutable data and function composition.
functions | Reusable blocks of code that take inputs and return outputs.
objects | Instances of classes bundling data and behavior in object-oriented programming.
programming languages | Formal languages for instructing computers, spanning imperative, functional, and object-oriented paradigms.
R | Programming language and environment designed for statistical computing and graphics.
static | Describes typing checked at compile time, or variables/methods bound to a class rather than instances.
