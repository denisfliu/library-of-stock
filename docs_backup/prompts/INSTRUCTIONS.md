The goal for this project is to create an interface for you to help a Quizbowl player study. We will be working interactively to try to create this project in a way that will be conducive to extracting important information (ordered roughly by how much it appears). The workflow will be as follows.

1. The user will input a query on a specific item of interest (could be person, philosopher, etc. etc).
2. We will search the qbreader database via their api which has documentation in qbreader_documentation.txt. We'll have to play around with how the api gives us information.
3. Once we figure that out, there are several ways in which the item of interest may appear. (1) As an answerline in a tossup or a bonus and (2) as text inside the tossup or bonus. For (1), the text of the tossup or bonus will serve as the clues that we might want to learn about the item of interest. For (2), the surrounding text might be involved. There are some exceptions, like sometimes a specific author's work might be clued but not mentioned explicitly but show up in the answer line as a side note: there might only be one or two sentences/clues that are relevant in this case.
4. With this in mind, we would like to output several key things. First, we want to build a summary of what the item of interest is with respect to how the quizbowl database defines it. That means we don't rely on outside clues at all; we only consider what the clues define item to be. Second, we want to indicate, in order of most common to least (approximately), the most "important" clues; this will entail grouping together clues that are essentially the same (note that clues are going to be worded differently in different contexts). With respect to authors, composers, and the like, this includes listing out their famous works and the most common clues corresponding to those works. We would like to output this in a way that is useful to read and learn from; so this means stating a work (or maybe the item of interest is a work), giving in order the clues from common to least common, and giving examples of what that would look like by e.g. pasting the clue from the qbreader database. For visual works, this would mean pasting a picture of the painting in, auditory, sound clips or something or scores. This should be visualized in a relatively intuitive manner, or maybe we can click on things and have them appear or something. Finally, we want this to also be conducive to do outside reading, so we should hyperlink important things and provide maybe relevant resources like relevant articles at the bottom.
5. We'll work on this later but we might eventually also automate the creation of anki cards using this.

Ultimately, we're creating a very tight workflow on learning about an item of interest in the context of quizbowl. With the exception of linking and pulling pictures, you should not be providing any outside information. The way we will work is up to you, but we should make it such that this is easy to run. You are also free to decide how the output is presented. We do want to have things presented in similar ways, so we can have some part of the workflow be coded, and some other part of it will need to be passed directly into an llm (you). So you are also free to propose different ways in which we can do this integration.
